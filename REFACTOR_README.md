# KeyLock 重构说明

## 重构概述

原有的 KeyLock 项目是一个单文件实现，包含了所有功能逻辑。为了提高代码的可维护性、可读性和符合 C++ 规范，已将代码重构为面向对象的多文件结构。

## 新代码结构

### 核心类

1. **Constants.h/cpp** - 常量定义和枚举
   - 定义了锁定状态、按键类型、资源ID、消息ID等常量
   - 移除了全局宏定义，改为类型安全的枚举和constexpr常量

2. **IndicatorContext** - 指示器上下文类
   - 封装单个键盘指示器的所有状态和配置信息
   - 管理HUD显示、托盘图标、音效等功能

3. **ImageManager** - 图像管理器
   - 负责加载和管理HUD图像和托盘图标图像
   - 支持从文件和资源加载图像

4. **TrayIconManager** - 托盘图标管理器
   - 创建、显示和清理托盘图标
   - 处理托盘图标的生命周期

5. **SoundManager** - 音效管理器
   - 播放系统音效和自定义音效文件
   - 支持Beep音效和MCI音效

6. **ConfigurationManager** - 配置管理器
   - 加载和解析配置文件
   - 初始化指示器上下文参数

7. **KeyboardHook** - 键盘钩子管理器
   - 安装和卸载键盘钩子
   - 处理键盘事件并分发到相应的指示器

8. **AutoStartManager** - 开机启动管理器
   - 检查、设置和取消开机启动
   - 管理注册表操作

9. **ResourceManager** - 资源管理器
   - 从资源文件中提取文件到磁盘
   - 处理配置文件和图像资源的提取

10. **MainWindow** - 主窗口类
    - 负责窗口创建、消息处理
    - 协调各个管理器的初始化和运行

11. **Application** - 应用程序类
    - 应用程序的主入口点
    - 管理GDI+初始化和单实例检查

### 设计原则

- **单一职责原则**: 每个类负责一个明确的功能模块
- **依赖注入**: 通过构造函数注入依赖，避免全局状态
- **RAII**: 使用智能指针和RAII管理资源
- **类型安全**: 使用枚举替代宏定义，使用constexpr常量
- **封装**: 将相关数据和方法封装在相应的类中

### 向后兼容性

- 保留了原有的全局函数声明以确保向后兼容
- KeyLock.cpp 现在作为应用程序入口，调用新的Application类
- 资源ID和消息ID保持不变

### 代码规范改进

- 移除了C风格的宏定义，改为C++风格的枚举和constexpr
- 使用了现代C++特性，如智能指针、引用等
- 改善了错误处理和资源管理
- 添加了详细的注释和文档

## 构建说明

项目使用Visual Studio 2019+ 构建。所有新文件已添加到项目文件中，编译时会自动包含所有必要的头文件和源文件。

## 运行说明

重构后的程序功能与原版完全一致，只是代码结构更加清晰和易于维护。
